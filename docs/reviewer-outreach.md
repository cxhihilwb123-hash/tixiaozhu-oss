# Reviewer Outreach Pack

Use this when inviting real reviewers to look at the repository. Do not ask for fake stars or copy-paste praise. Ask for concrete, honest feedback.

Repository:

```text
https://github.com/cxhihilwb123-hash/tixiaozhu-oss
```

## Message for an Engineer

```text
Hi, I published Tixiaozhu as an AGPL open-source Chinese AI education-product engineering sample. Could you clone it, try the Quick Start, and open one honest issue if anything is confusing or broken?

I especially want feedback on README clarity, local setup, API structure, CI, and whether the launch-gate design is useful for contributors.
```

## Message for an Education / Parent Reviewer

```text
Hi, I published Tixiaozhu as an AGPL open-source education-product sample. Could you look at the README screenshots and question-store structure and tell me whether the product logic makes sense for primary-school Chinese/math/English practice?

Honest feedback is better than praise. If something is unclear, please open an issue or tell me what should be improved.
```

## Message for a Backend / Security Reviewer

```text
Hi, I published Tixiaozhu as an AGPL open-source full-stack education-product sample. Could you review the production-readiness posture, auth scoping, payment deferral, and commercial launch gates?

If you find a security issue, please report it privately according to SECURITY.md. For normal improvement ideas, opening a GitHub issue would help the project.
```

## What Reviewers Can Do

- Open one issue with setup feedback.
- Open one issue with docs feedback.
- Open one issue with product/content feedback.
- Comment on the AGPL/commercial licensing FAQ if it is unclear.
- Submit a small PR for docs wording or setup clarification.

## What Not To Ask For

- Do not buy stars.
- Do not ask for fake reviews.
- Do not ask people to say they use the project if they did not.
- Do not overclaim adoption, production usage, or commercial readiness.
